  <name>Task 2: Create PDF parse API route with tests</name>
  <files>src/app/api/parse/route.ts, src/__tests__/api-parse.test.ts</files>
  <behavior>
    - POST /api/parse accepts multipart form data with a "file" field containing a PDF
    - Returns 400 if no file provided
    - Returns 400 if file is not a valid PDF (magic byte check: first 4 bytes must be %PDF / 0x25504446)
    - Returns 400 if file exceeds 5MB
    - Returns 422 with scanned:true and friendly message if extracted text is less than 20 non-whitespace chars
    - Returns 200 with { text: string, pages: number } on successful extraction
    - Uses pdfjs-dist server-side for text extraction (legacy build path for Node.js compatibility)
  </behavior>
  <action>
    1. Install pdfjs-dist: `npm install pdfjs-dist`

    2. Create `src/app/api/parse/route.ts`:
       - Export async POST handler accepting Request
       - Parse formData: `const formData = await request.formData()`
       - Get file: `const file = formData.get('file') as File | null`
       - Validate: return 400 if no file
       - Size check: return 400 if `file.size > 5 * 1024 * 1024` ("File exceeds 5 MB limit")
       - Convert to buffer: `const buffer = new Uint8Array(await file.arrayBuffer())`
       - Magic byte check: verify first 4 bytes are 0x25, 0x50, 0x44, 0x46 (%PDF). Return 400 if not.
       - Dynamic import pdfjs-dist: `const pdfjsLib = await import('pdfjs-dist/legacy/build/pdf.mjs')`
       - Load document: `const doc = await pdfjsLib.getDocument({ data: buffer }).promise`
       - Extract text from each page: iterate 1..numPages, getPage, getTextContent, join items
       - Scanned detection: if total non-whitespace chars < 20, return 422 with friendly error message per CONTEXT.md: "This PDF appears to be a scanned image. Try a text-based PDF or paste your resume as plain text."
       - Return 200 with `{ text: fullText, pages: doc.numPages }`
       - Wrap in try/catch: return 500 on unexpected errors

    3. Write `src/__tests__/api-parse.test.ts`:
       - Test the route handler validation logic (no file, oversized file, invalid magic bytes)
       - Mock pdfjs-dist for unit testing (avoid needing real PDF files in test)
       - Test scanned PDF detection (mock getTextContent returning empty items)
       - Test successful extraction (mock getTextContent returning text items)
       - Use `vi.mock` for pdfjs-dist module

    NOTE: The pdfjs-dist import path may need adjustment if `pdfjs-dist/legacy/build/pdf.mjs` doesn't resolve correctly in Next.js. If so, try `pdfjs-dist` directly and set `GlobalWorkerOptions.workerSrc` to empty string to disable worker. Test the import during implementation and adjust accordingly.
