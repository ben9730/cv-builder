2. Install runtime dependencies:
   `npm install zustand zod`

3. Install dev dependencies for testing:
   `npm install -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/dom vite-tsconfig-paths`

4. Verify postcss.config.mjs uses `@tailwindcss/postcss` (Tailwind v4 pattern). If create-next-app generated a different config, replace with:
   ```javascript

     plugins: {
       "@tailwindcss/postcss": {},
     },
   };
   export default config;
   ```

5. Verify globals.css uses `@import "tailwindcss"` (Tailwind v4 single-import pattern, not the old three-line @tailwind directives).

6. Verify tsconfig.json has `"strict": true` in compilerOptions. If not, add it.

7. Replace the default page.tsx content with a minimal placeholder:
   ```tsx
   export default function Home() {
     return (
       <main className="flex min-h-screen items-center justify-center">
         <h1 className="text-3xl font-bold">CV Builder</h1>
       </main>
     )
   }
   ```

8. Ensure the root layout.tsx is clean (no unnecessary boilerplate like font imports beyond the default).

IMPORTANT: Do NOT create a tailwind.config.js or tailwind.config.ts — Tailwind v4 uses CSS-first config via @theme directives in globals.css. A JS config file will be silently ignored.
