  <action>
1. Install docx library (if not already installed from Plan 01):
   ```bash
   npm install docx
   ```

2. Create `src/components/export/docx-export.ts`:
   - Import { Document, Packer, Paragraph, TextRun, HeadingLevel, AlignmentType, BorderStyle } from 'docx'
   - Import { saveAs } from 'file-saver'
   - Export async function `exportDocx(resume: ResumeData)`:
   - Build a single-section Document with structured content:
     - **Header:** Name as Heading 1, label/title as subtitle, contact info (email, phone, location, url) as a paragraph
     - **Summary:** If present, as a normal paragraph
     - **Experience:** "Experience" as Heading 2, then for each work entry: company + position as bold text with dates, bullet list from highlights array
     - **Education:** "Education" as Heading 2, then for each entry: institution + degree/area, dates, GPA if present
     - **Skills:** "Skills" as Heading 2, then for each skill: category name bold with keywords as comma-separated text
     - **Optional sections** (certificates, projects, languages, volunteer): Only if defined and non-empty. Each gets a Heading 2 and formatted entries.
   - Use proper DOCX formatting: HeadingLevel.HEADING_1, HeadingLevel.HEADING_2, bold TextRuns, bullet lists via Paragraph bullet option
   - Do NOT try to replicate two-column layout (Modern template) in DOCX — DOCX export focuses on structure and readability (per research pitfall #6)
   - Generate blob: `const blob = await Packer.toBlob(doc)`
   - Save: `saveAs(blob, '${resume.basics.name || "resume"}.docx')`

3. Create `src/components/export/json-export.ts`:
   - Import { saveAs } from 'file-saver'
   - Export function `exportJson(resume: ResumeData, selectedTemplate: string)`:
     - Build backup object: `{ version: 1, exportedAt: new Date().toISOString(), template: selectedTemplate, resume }`
     - Per CONTEXT.md: JSON backup includes resume data + settings (template choice)
     - Serialize: `JSON.stringify(data, null, 2)`
     - Create Blob with type 'application/json'
     - Save: `saveAs(blob, '${resume.basics.name || "resume"}-backup.json')`

4. Create `src/__tests__/docx-export.test.ts`:
   - Mock 'docx' module: mock Document constructor, Packer.toBlob returning Promise<Blob>
   - Mock 'file-saver' saveAs
   - Test that exportDocx creates a Document and calls Packer.toBlob
   - Test that saveAs is called with a blob and .docx filename
   - Test filename fallback when name is empty
   - Test that optional sections are included only when defined

5. Create `src/__tests__/json-export.test.ts`:
   - Mock 'file-saver' saveAs
   - Test that exportJson calls saveAs with a JSON blob
   - Test that the JSON content includes version, exportedAt, template, and resume fields
   - Test that template selection is included in the backup (PERS-02)
   - Test filename has '-backup.json' suffix
   - Test filename fallback when name is empty
  </action>

    - Both functions use file-saver for cross-browser download
    - DOCX focuses on structured content (not visual template matching)
    - JSON backup includes version, timestamp, template preference, and full resume data
    - All tests pass with mocked dependencies
